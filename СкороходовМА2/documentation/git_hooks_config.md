# Конфигурация Git Hooks

## Инструмент

Для локальных Git hooks используется `pre-commit`.

Версия:

```text
pre-commit 4.6.0
```

Hook установлен командой:

```bash
pre-commit install
```

После установки перед каждым `git commit` автоматически запускаются проверки из `.pre-commit-config.yaml`.

## Список хуков

| Hook | Тип | Что проверяет | Польза |
|---|---|---|---|
| `trailing-whitespace` | formatting | Пробелы в конце строк | Уменьшает шум в diff и предотвращает неаккуратные изменения |
| `end-of-file-fixer` | formatting | Пустую строку в конце файла | Делает файлы совместимыми с Unix tooling |
| `check-yaml` | syntax | Корректность YAML | Защищает CI/CD и compose-конфиги от синтаксических ошибок |
| `check-json` | syntax | Корректность JSON | Помогает не коммитить битые JSON-файлы |
| `check-added-large-files` | repository hygiene | Случайно добавленные большие файлы | Предотвращает попадание тяжелых артефактов в Git |
| `detect-secrets` | security | Потенциальные секреты в коде | Снижает риск утечки токенов, паролей и ключей |
| `gofmt` | formatting | Форматирование Go-кода | Поддерживает единый стиль Go-кода |
| `go vet` | lint/static analysis | Подозрительные конструкции Go | Находит часть ошибок до запуска приложения |
| `go test` | tests | Unit-тесты проекта | Блокирует коммит при сломанной бизнес-логике |

## Особенности настройки секретов

В проекте есть учебные локальные значения:

- `postgres://postgres:postgres@localhost:5432/gotaskflow`;
- `POSTGRES_PASSWORD=postgres`.

Они используются только для локального Docker Compose и помечены inline-комментариями `pragma: allowlist secret`. Production-секреты так хранить нельзя: их нужно задавать через GitHub Secrets, `.env` вне Git или secret manager.

## Инструкция для нового контрибьютора

1. Установить Python 3.13 или совместимую версию.
2. Установить `pre-commit`:

```bash
python -m pip install pre-commit
```

3. Установить Git hook в репозитории:

```bash
pre-commit install
```

4. Проверить все файлы:

```bash
pre-commit run --all-files
```

5. Перед отправкой изменений дополнительно запустить:

```bash
go test ./...
```

## Пример блокировки плохого коммита

Если в Go-файле нарушено форматирование или в YAML есть ошибка, `pre-commit` завершится с ошибкой и Git не создаст commit. Разработчик должен исправить проблему, добавить исправленные файлы через `git add` и повторить commit.
